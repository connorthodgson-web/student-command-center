# Agent Instructions

These instructions are for coding agents working on Student Command Center.

## Product Understanding

This project is an assistant-first student app.

It should not be treated like a generic planner template.
It should not be treated like a productivity dashboard with AI added later.

The core product experience is:
- students can talk naturally to the system
- the system helps organize and understand academic life
- the assistant experience is more important than complex planner features

## Coding Principles

- Keep the project beginner-friendly
- Do not overengineer
- Prefer small iterative changes
- Use clear naming
- Preserve working code
- Keep files readable
- Avoid unnecessary abstractions
- Explain changed files clearly

## MVP Priorities

Prioritize:
- natural-language input
- AI chat
- class/task data structure
- reminder settings
- clean simple UI

Deprioritize:
- advanced urgency ranking
- unnecessary animations
- dashboard complexity
- speculative future systems

## Implementation Style

When building features:
- keep components practical
- keep API routes clear
- add TODOs where future AI or backend work is expected
- avoid pretending unfinished systems are complete
- prefer placeholders over fake complexity

## Communication

At the end of each implementation task:
- summarize what changed
- list files changed
- list TODOs or next steps
